# Quick Start - Smart Discovery Mode

## Your Use Case

- **Current:** 1 project (taleemabad-core)
- **Goal:** Easy to add more projects
- **LLM:** Start with Ollama, switch to Claude if needed
- **Custom Rules:** Use `.github/claude.md` in each project

---

## 🚀 3-Minute Setup

### 1. Update Current Project Structure

```bash
# Add a claude.md file to your existing project
cd /path/to/taleemabad-core
mkdir -p .github
```

Create `.github/claude.md`:
```markdown
# Taleemabad Core - Review Standards

## Critical Rules

### 1. Soft-Delete Only
Never use `.delete()` - always set `is_active = False`

### 2. Multi-Tenancy Safety  
All queries must respect tenant boundaries

### 3. Migration Safety
Use `apps.get_model()` in migrations, never direct imports

[... your existing rules from pr-review-bot/guides/ ...]
```

### 2. Simplified Config

```yaml
# config/projects.yaml
projects:
  - name: taleemabad-core
    repo: Orenda-Project/taleemabad-core
    
    # LLM Configuration
    llm:
      provider: ollama              # Start with free Ollama
      model: llama3.2
      base_url: http://localhost:11434
      # If switching to Claude later:
      # provider: anthropic
      # model: claude-sonnet-4
      # api_key: ${ANTHROPIC_API_KEY}
    
    # Smart Discovery
    discovery:
      auto_detect: true             # Auto-detect Django + React
      guide_files:                  # Load project-specific rules
        - .github/claude.md
        - pr-review-bot/guides/CODE_REVIEW_GUIDE.md
        - pr-review-bot/guides/TEST_REVIEW_GUIDE.md
    
    # Review Settings
    review:
      max_comments: 8
      auto_approve_on_resolution: true
      generate_improvements: true   # Create improvement reports
```

### 3. Run First Discovery

```bash
# Bot analyzes your codebase
pr-review-bot discover --project taleemabad-core

# Output:
# ✓ Detected: Django 4.2, React 18, PostgreSQL
# ✓ Loaded custom guide: .github/claude.md
# ✓ Generated comprehensive review guide
```

### 4. Start Reviewing

```bash
# Review all open PRs
pr-review-bot review --project taleemabad-core

# Or review specific PR
pr-review-bot review --project taleemabad-core --pr 123
```

## 🎯 Adding Your Second Project

When you're ready to add another project:

```yaml
# config/projects.yaml
projects:
  - name: taleemabad-core
    # ... existing config ...
  
  - name: new-project                    # Just add these lines!
    repo: myorg/new-project
    llm:
      provider: ollama
      model: llama3.2
    discovery:
      auto_detect: true                  # Bot figures out the stack
      guide_files:
        - .github/claude.md              # Project-specific rules
```

Then:
```bash
# Bot automatically:
# 1. Analyzes the new codebase
# 2. Detects frameworks (Django/Next.js/React/etc)
# 3. Loads your custom claude.md
# 4. Starts reviewing PRs

pr-review-bot discover --project new-project
pr-review-bot review --project new-project
```

**Time to add new project: 5 minutes!**

## 📝 Example .github/claude.md Templates

### Django Project
```markdown
# Project Review Standards

## Critical Rules
- Soft-delete only (no `.delete()`)
- Multi-tenancy safety
- Migration safety (use `apps.get_model()`)

## Security
- All endpoints require authentication
- Validate input with serializers
- No secrets in code

## Testing
- Use factory-boy, not `.objects.create()`
- Tests in `tests/` directory
- 80% coverage required

## Patterns
- ViewSets for CRUD
- Celery for async tasks
- Redis for caching
```

### Next.js Project
```markdown
# Next.js Review Standards

## Critical Rules
- Server Components by default
- Add 'use client' only when needed
- All images use next/image
- All pages have metadata

## Performance
- Implement loading.tsx
- Use Suspense boundaries
- Optimize fonts with next/font

## API Routes
- Validate input with Zod
- Return proper status codes
- Rate limit public endpoints
```

### FastAPI Project
```markdown
# FastAPI Review Standards

## Critical Rules
- Pydantic models for validation
- Async endpoints where possible
- Dependency injection for DB sessions

## Security
- JWT authentication
- Input validation
- SQL injection prevention

## Testing
- pytest with async support
- Test client for API tests
- Mock external services
```

## 🆘 Quick Troubleshooting

### Ollama not responding
```bash
# Check if Ollama is running
curl http://localhost:11434/api/tags

# Start Ollama
ollama serve

# Pull model
ollama pull llama3.2
```

### Bot doesn't detect frameworks
```bash
# Run discovery in verbose mode
pr-review-bot discover --project taleemabad-core --verbose

# Manually specify frameworks if needed
# In config/projects.yaml:
discovery:
  frameworks: [django, react]  # Override auto-detection
```

### Review quality is low with Ollama
```bash
# Switch to Claude
# Edit config/projects.yaml:
llm:
  provider: anthropic
  model: claude-sonnet-4
  api_key: ${ANTHROPIC_API_KEY}

# Export Claude key
export ANTHROPIC_API_KEY=sk-ant-...

# Try again
pr-review-bot review --project taleemabad-core --pr 123
```
